I see a few posts about some of these topics already, but I feel these things need to be reiterated.

This is by far my favorite distro so far. For some reason I just click with it. I cannot entirely explain why. However, stock antiX is not very friendly to laptop users.

I just converted from Protech ONE (an ubuntu feisty fluxbox security based distro). The developers of antiX may want to take a look at Protech, which while not being as good as antiX has a few things that could be implemented in antiX.

Just a few comments, but I am just another peon programmer in training.
  • + wicd or some other wireless manager needs to be installed by default. Protech uses wicd (I know this is not Protech and I am glad it is not).

Firstly welcome to antiX, Urban.

I'll take another look at Protech ONE when I get the chance.

1. wicd will almost certainly be installed by default in the next antiX.

2. dhclient 'problem' has been fixed for next release.

3. About the conkyrc script- good idea!

4. The problem with having an automatic update of the fluxbox menu in Debian is that it will use the Debian fluxbox menu configuration, which IMO is pretty unintuitive. Using update-menu does work in antiX, but you would have to edit the /.fluxbox/init file.

5. Auto-removing those entries is being worked on. There is a manual way to do this. See this thread.
