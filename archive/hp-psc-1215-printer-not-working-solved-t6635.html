My HP psc 1215 printer does not work. I have installed the metapackages for printing and scanning, also the 'Install Printer' grafical interface (it does however not seem to work 100% properly). The grafical interface deny access to installing more printers, even as Root. (It names the printer PDF.)
I use AntiX Base (non-pae, so-called i386 version), on an i686 (HP ze4913ea) with 512 MB RAM.
Previously I used this printer on PCLinuxOS lxde (now discontinued) for six years without any problem (until that system crashed on Sept, 23rd).
The CUPS on that lxde system installed a driver file when plugging in the printer, AntiX does not.
(I have done an error search, but for some reason the (very long) textfile cannot be pasted into this forum.)
What to do ?
lsusb gives: Bus 001 Device 007: ID 03f0:2f11 Hewlett-Packard PSC 1200
'HPsetup' recognizes printer HP psc 1200, but deny access to 'add printer', do not accept Root password !!!
How to get 'HPsetup' to accept password ???
SOLVED: 1. Uninstalling 'Install Printer' grafical interface. 2. Disabling autologin (chosen when installing from CD, it has some kind of bug, substitutes chosen administrator password with chosen user password. 3. Confirming the passwords in the 'Users' grafical interface. 4. Running HPsetup.
