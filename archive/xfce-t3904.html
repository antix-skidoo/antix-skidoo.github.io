To know if you have consolekit installed I think you need
apt-cache search consolekit

then look for the packages and run (to see if they are installed)
apt-cache show <put here the package name>

I'm on fedora so I don't remember the details (we use rpm/yum).

Then if you have it installed, fine, otherwise install it.
Then check your .xinitrc file in your home folder to see if it has a command line to start xfce and use the one I posted the other post.

This is as far as I can go because antix has a customized way of starting sessions and that's why it sometimes doesn't work out-of-the-box as Debian starts the sessions differently by default.

rokytnji's way will work to shutdown, but I believe you are refering to the shutdown and reboot options being greyed out in xfce?

I have had this problem with xfce on antix, to fix you need to do what secipolla says. Install consolekit..
menu --> run
type: gksu roxterm
press: ok
type: YOUR ROOT PASSWORD
press: ok
type: apt-get update && apt-get -f install consolekit
press: enter/return
wait for prompt and type: y
press: enter/return
wait till it is finished.

The second part secipolla says is to add a line to ~/.xinitrc to start consolekit. However I believe slim is now setup to start consolekit in the same way, (all I have had to do is install consolekit) so it may not be needed.
