Using GRUB 2, there is a variable called GRUB_TIMEOUT that defaults to 5 - five seconds. It can be overridden in the file /etc/grub.d/00_header. Using GRUB Legacy, in case anyone is using it, there is a similar value, simply called timeout, and it is often set to 5. This parameter can also be changed as needed

syntax

timeout 30

gnomic wrote: How does one boot in RAM by the way?
Add the boot argument toram to the boot screen (press Tab to allow entry of additional arguments or just add them to the end of the kernel line of the boot record, for example, for this line:

linux /boot/vmlinuz-2.6.37-4.dmz.1-liquorix-686 root=UUID=05d17e98-d50d-4a54-99c5-ce7a463229a4 ro

make it

linux /boot/vmlinuz-2.6.37-4.dmz.1-liquorix-686 root=UUID=05d17e98-d50d-4a54-99c5-ce7a463229a4 ro toram

Sometimes copy2ram also works, and when in doubt, I throw in both; one or the other will"take". In the case of antiX, according to
