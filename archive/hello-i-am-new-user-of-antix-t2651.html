Welcome to antiX, the OP had dl the antiX 8.5 alpha 2 version, that was a testing version, and due to the rolling release nature of antiX, upgrading would entail a very large number of updates, theoretically speaking, had the OP dl the alpha2 at the time of the release of the alpha2, it would have been for test purposes, to ferret out the bugs and following that logic, he would not have been using this for a production rig. He would have probably dl the final and then tweaked to his likings and done an apt-get update && apt-get dist-upgrade on a regular basis, which would entail a fair number of updates. But for someone who just dl'd the latest version, (July) would only necessitate installing updates since July.
Back to your question, had the OP installed the original final and done a du on a regular basis, he would be up to date and would not need to install the July release, that was sort of a maintenance release that would allow new users to dl the latest up to date version.

Have a read here if you have not already, it will enlighten you to some of the possibilities of antiX and again welcome.
