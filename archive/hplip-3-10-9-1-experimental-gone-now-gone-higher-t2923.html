Re hplip 3.10.9-1, currently in debian experimental

As a test, which worked, using antix-M11-pt1:

-installed hplip and dependencies from debian squeeze

-then used dpkg -i to use only, only, the following from debian experimental (as a sort of upgrade)

hplip_3.10.9-1_i386.deb
libhpmud0_3.10.9-1_i386.deb
hplip-cups_3.10.9-1_i386.deb
libsane-hpaio_3.10.9-1_i386.deb
hplip-data_3.10.9-1_all.deb

These are the packages that require an exact version match.
Maybe use dpkg --configure –a afterwards.
Then, if you like, try the hplip-gui stuff, same strategy.

Oh, ghostscript-x wanted an upgrade after this.
