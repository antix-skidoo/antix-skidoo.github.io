This is what, I do to fix missing, and broken dependence files.

run first as root.

Code: Select all

Code: Select all

Code: Select all

apt-get --fix-broken install
,and or

Code: Select all

apt-get install --fix-missing

Thanks! The issue is solved!
(In the following I describe the details.)

@ anti: under X the only terminal availiable from the menu is the root-terminal.
I opened it and su to username.
indigo@T160i:~
$ lxappearance
No protocol specified Error: could not open display.

Then I exit user, and beeig root again. lxappereance opened up fine.
I choose MediterranianNight, apply, close.
geany opens up, rhythmbox and terminal not. => no improvement.
logout - login makes no changes.
I repeated the procedure above with clearlooks. (without beeing user first...)
No changes - rhythmbox and terminal do not open up.

__{{emoticon}}__ Yes, I used the testing-repos (without beeing aware...).
I changed them to wheezy/stable manually (using synaptic), ran into unsolvable dependencies.
I restarted synaptic again and updated the proposed packages.
Then I forced especially the libgtk3-x and libgtk2.x packages to the stable version.
Synaptic did not like to install rhythmbox, so I deinstalled it.
I applied the changes.
Since then calling terminal from menu worked again!

On console I su to root, then doing dpkg --configure -a.
and

Code: Select all

apt-get install rhythmbox
everything seems to work now!

Thanks for your help, I consider the issue to be solved.
The crunch was to define the right repo. __{{emoticon}}__
Thanks a lot !!!
