Aha! I believe I have fixed the problem.

Found this:
========= SCRAPER REMOVED AN EMBEDDED LINK HERE ===========
url was:"http://bugs.debian.org/cgi-bin/bugreport.cgi?bug=638586"
linktext was:"http://bugs.debian.org/cgi-bin/bugreport.cgi?bug=638586"
====================================

Which gave me the previous version of libgnutls26 (2.10.5-3).

Found this:
========= SCRAPER REMOVED AN EMBEDDED LINK HERE ===========
url was:"http://www.crazysquirrel.com/computing/debian/downgrade.jspx"
linktext was:"http://www.crazysquirrel.com/computing/ ... grade.jspx"
====================================

Which walked me through finding the cached packages, and using dpkg.

At first, dpkg -i libgnutls26_2.10.5-3_i386.deb gave me an error saying that the file didn't exist. My meager linux knowledge was thankfully enough to puzzle-out that I needed to cd to /var/cache/apt/archives and *then* run dpkg -i libgnutls26_2.10.5-3_i386.deb.

That worked. Claws Mail slowly chugged to life, and downloaded a crapload of email.

Now all I need to figure out is how to tell apt to hold libgnutls26 from upgrading. I'm sure I saw a how-to on one of the posts I scanned through in the past couple of days...

Which listed a clever little command line way to hold a package:

I received no error messages when I used it, so perhaps it worked __{{emoticon}}__
