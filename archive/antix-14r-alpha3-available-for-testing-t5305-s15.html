Symlinking is in place for the menu update. Here is a follow through of the process.
1 menu is symlinked to global position in /usr/share/desktop-menu/.WM/menu-applications
2. apt-get is run
3 apt refers to / etc/apt/apt.conf.d/ for various scripts and runs 99-update-menus
4 99-update-menus refers to /usr/local/bin/desktop-menu-apt-update set
5. this records the modified times of /usr/share/applications
6 99-update-menus refers to /usr/local/bin/desktop-menu-apt-update check
7 this compares the modified time recorded and the current modified time of /usr/share/applications.
8 if different /usr/local/bin/desktop-menu-apt-update calls /usr/local/bin/desktop-menu to update the global menu. If the same quitely exits
9 if the global menus are updated it is automatically transferred to the user via the symlink

The key to make it work with dpkg / gdebi would be replace steps 3 with something like
3 dpkg refers to / etc/DPKGCONFIGDIR/ for various scripts and runs 99-update-menus
