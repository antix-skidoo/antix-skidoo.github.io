In my case was solved by running the full path command
/usr/lib/i386-linux-gnu/gdk-pixbuf-query-loaders
/usr/lib/i386-linux-gnu/gdk-pixbuf-2.0/2.10.0/loaders.cache and restarting
gdm
I searched for missing command there, missing symlink as they suggested, and in my case ran as root:

Code: Select all

/usr/lib/i386-linux-gnu/gdk-pixbuf-2.0/gdk-pixbuf-query-loaders > /usr/lib/i386-linux-gnu/gdk-pixbuf-2.0/2.10.0/loaders.cache
Restarted machine (because restarting JWM did not do anything, so I just took the violent, more direct approach), and voila! Icons appeared.

And mirage now runs with no messages on commandline.
