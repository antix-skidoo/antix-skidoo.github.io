    I think this is related to the wallpaper app not opening not the control center.
    In this aspect try running wallpaper.py from terminal and posting the output.
    Chances are that the configuration file has a line linking a picture that has been removed.
    This happens when you select a picture for the wallpaper, then use the file manager to (re)move the picture, then try resetting the wallpaper with a new picture but cannot as the wallpaper app cannot find the old picture. You can fix this by editing ~/.antix-session/wallpaper-list.conf to point to an existing picture.

    It is something that I have been looking into fixing for 14. Currently I have 2 solutions to the problem and I am debating which one would be better.
    1) make a copy of the picture set in ~/.antix-session (soon to be desktop-session). This requires one copy of the picture for every available session but you will not see an ugly default wallpaper as with the next solution.
    2) Before trying to open the wallpaper app, verify if the picture recorded exists or not. This will be the"leanest" but will also result in your wallpaper being set to a standard hard coded one.
