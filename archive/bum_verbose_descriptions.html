Human-Readable Descriptions of init Scripts

Many of the services launched via init scripts in runlevels 2-3-4-5 and rcS.d lack standard, "human readable" descriptions of their main behavior. Often, the package description provided for these services is very cryptic and (and, cryptic or not) longer descriptions cannot easily be displayed within the BootupManager main view window.

Confusingly, the name of a given service may not exactly match the debian packagename which provided (installed) it.
Example: "motd" (message of the day) 
$ locate motd
  ^---v
/etc/motd
/etc/update-motd.d
/etc/init.d/motd

$ dpkg-query -S /etc/init.d/motd
  ^---v
initscripts: /etc/init.d/motd

$ apropos motd
  ^---v
motd (5)             - message of the day
pam_motd (8)         - Display the motd file
