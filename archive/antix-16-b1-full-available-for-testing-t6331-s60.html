Idea for Meta Package Installer

This opinion was expressed previously in this thread. It succinctly encapsulates similar expressions made around the same time.
masinick wrote:Add me to the"camp" that generally prefers modest resource use. We have a reasonable blend between simplicity, ease of extending, maybe not optimal for the absolute novice, but that's not really our top focus group - those that need efficient resource utilization probably are on the top of the priority list...
At the OS level antiX is undoubtedly efficient. That does not automatically extend into the application level. Optimum efficiency and performance requires partnering the OS with appropriate lightweight apps. Some of the apps currently in Meta Package Installer (MPI) may defeat that goal.

Having apps within MPI can be seen as conferring the distro seal of approval on each of them. It suggests they all work well on any kit that runs the OS. With antiX this is only partially true, because it targets both modern highly spec'd kit and old lowly spec'd kit. In simple terms, using MPI in its current form can produce disappointing results for antiX users.

To a user, when installing a new app, the primary consideration is, will this enable me to do what I want/need to do? To an antiX user, the next priority is will it run OK on my kit? The latter is an opportunity to become more user friendly.

Within its initial interface window, MPI might offer generalized, non proscriptive guidance. Each app might be pre-allocated into one of three categories, and the guidance shown in key form something like:
[12] Works on kit manufactured in the last 12 years and possibly older
[7] Best suited to kit manufactured in the last 7 years
[3] Best suited to kit manufactured in the last 3 years
Expressing the key in terms of the age of kit is likely to be more immediately understandable than terms such as lightweight, medium-weight, heavyweight.

Each individual app will display the reference to the key as the very first item in its description area
Example: in the current Graphics section of MPI is ImageMagick. Its description is presently
ImageMagick - a software suite to create, edit and compress bitmap images
It will become
[12] ImageMagick - a software suite to create, edit and compress bitmap images

In this way the user is able to make a more informed choice before installing via MPI.
