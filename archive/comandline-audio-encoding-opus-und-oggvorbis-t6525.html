    Zum Encodieren müssen die OPUS- und die OGG-Vorbis-Tools installiert sein.
    Bei meinem Remaster ist das der Fall.


    Aus Wav- oder FLAC-Dateien OPUS- oder Ogg-Vorbis-Dateien erstellen

    Terminal öffnen
    opusenc X.wav X.opus (96kbit/sec VBR / 86.7164kbit/s)
    oggenc X.wav X.ogg (Average bitrate: 96,4 kb/s)
    oggenc X.wav -q 5 X.ogg (Average bitrate: 138,8 kb/s)

    X.wav muss im BenutzerVerzeichnis liegen.


    Aus OPUS- oder Ogg-Vorbis-Dateien WAV-Dateien erstellen

    Terminal öffnen
    opusdec X.opus X.wav
    sox X.ogg X.wav

    Qmmp 0.8.2 kann bei mir trotz Opus-Decoder (libopus.so)
    keine Opus-Dateien abspielen. Der SMPlayer aber kann das.


    Aus Soundtracker tunes Wav-Dateien erstellen

    Terminal öffnen
    mikmod
    Configuration>Driver Wave disk writer
    Save config
    Use config
    Other options> Playmode >Loop list deaktivieren
    File>Load
    X.it
    erstellt music.wav
    Source file verschieben oder umbenennen,
    da die Einstellungen nicht automatisch zurückgesetzt
    werden und bei jedem Start von mikmod erneut
    die Wav-Datei erstellt wird.


    Qualitäts-Vergleich bei um die 60 kbps

    opusenc --bitrate 60 RCP23.wav RCP23.opus
    oggenc RCP23.wav -q 0 -m 32 -M 112 RCP23.ogg


    RCP23.zip (RCP23.7z - RCP23.it, RCP23.opus, RCP23.ogg)
