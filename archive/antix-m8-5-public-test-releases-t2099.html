Ok, I booted in English to fluxbox and set my keyboard with '# setxkbmap br'.

First of all, thank you, anti, the applications sub-menu is an (I would say essential) improvement. ICE, for you to understand: the applications sub-menu we have up to aX-8.2 is derived from Debian's menu and generated from the menu files in /usr/share/menu (through the package 'menu'). Some applications don't follow this system and as such don't appear inside the menu after installed. They use /usr/share/applications instead (which has .desktop files instead of menu files). This is a standard for Gnome, Xfce and LXDE, for example and I believe is the standard for the freedesktop foundation. So they appear in the sub-menus according to their settings in the .desktop files. If they appear in wrong places it's a bug with the app (although pretty easy to fix in the system by editing its .desktop file). PcManFM appears under accessories and system tools (as you may see in Xfce's menu too).
anti, in keeping this there's no need anymore for the 'Debian menu' help.

The problem ICE had with the screenshot app is because it's named antixscreenshot-en.sh and in the keys file there is antixscreenshot.sh. Its shortcut is still Alt+S.
