In cases where a swap area exists in both RAM (zram) and on disk, does the system have a preference for one or the other?

For example, does it use (say) the RAM swap until it is exhausted then begin to use the disk swap, or does it use either of them as it see fit without preferring one over the other?

SamK wrote:For example, does it use (say) the RAM swap until it is exhausted then begin to use the disk swap [...]
Yes, it does it that way. You always want to use the faster resource first:
  • cpu registers,
  • cpu cache (increasing levels),
  • ram,
  • zram, and then finally
  • swap.
Generally, going down the list you have increasing size and decreasing speed but this is not always the case.
