Installed from DVD onto very old hardware to push the limits of antiX -- 1998 Toshiba Tecra 780dvd/cdm laptop (233MHz Pentium 2 CPU, 192MB RAM), here's what I happened for me (I'd love to see what other people are doing to get antiX on super-old comptuers):

- very slow boot ~169 seconds for initrd

- booted successfuly to standard GUI, so I tried to install from ICEWM desktop icon but it froze sometime shortly after 30%,

- tried to install again from min-jwm (because it uses a bit less RAM) but still froze.

- rebooted tried BitJam's idea of"fdb+ 3" and then used cli-installer (disabled bluetooth and cups services) and it took just under an hour (53mins) but it worked! I installed on an old 10GB HDD (4200rpm?) and did a 200MB swap file (since it has 192/180MB of RAM), next time I'll install on a 16gb mSATA, but to test the extreme I wanted to use the old hardware, and this HDD while old was actually newer than the comupter itself.

- when rebooted to GUI changed desktop to greyscale wallpaper, and did min-jwm desktop option --> ~46-53MB RAM out of 180MB showing up (out of the 192MB installed).

- used Control Center to start Synaptic and uninstalled FireFox since it is waaaaaay too RAM/CPU intensive for this old beast but uninstalling was glacially slow.

- tried changing the default application of the web browser (using the control center) to Dillo and Links2 but it wouldn't stick (i.e., it was blank when reopened it and there was no functioning default web-browser), maybe I should have done this before uninstalling FireFox, maybe removing it wrecked this, I dunno.

- like in my Thinkpad 600x test, the Sound Card (Yamaha 3D sound effect-enabled OPL3-SA3 = Sound Blaster 3.01) didn't work -- in fact it doesn't even show up in inxi ! Tried to test the sound using the Control Center, and same thing as with 600x a screen pops up very fast and can't see whatever error there might be.

- I modified the Grub files (as per eugen-b's advice in earlier posts) to boot in 3 for CLI option

- rebooted into cli . screensize was wrong (has 1024x768 screen) just showing as a centered smaller box in middle (maybe VGA 640x480?) it did this also when did the install from cli but I wrongly assumed it would be fixed after it installed, and it also shows this on the Grub screen too.

- I tried the antiX-cli-cc to fix this but when using the console-width-select got an error"is framebooter enabled", other options under that menu likewise failed to change the screen size or font color (which was different than in terminal from the GUI boot).

- RAM use under the cli boot was lower ranging from 33MB (env-info, htop, neofetch) to 40MB (screenfetch) to 44MB (inxi). This is good showing how little RAM antiX can run on (again the Tecra 780, while maxed out, still only has 192MB of RAM which antiX reports as 180MB)!

- To test installation under cli I installed ninvaders and it took a long 56.24 seconds. game played fine (well no sound as per above).

- Thinking about the sound I thought maybe it was a kernel issue not recognizing such an old sound system so I tried to install the Debian Fallback 32-bit Kernel. The cli-aptiX says"the 'kernels' feature isn't implemented yet" so I'll try that again under Beta2. I tried it under the GUI with min-jwm and it also froze (108/180MB RAM was last I saw in the conky).

- Ethernet worked automatically through DLink PCMCIA card!

- Overall, while it is possible to use a computer this old (again I'd love to compare notes with anyone else trying this on super-old computers) things are pretty slow and some applications are just too intensive to use at all (e.g., FireFox) or too slow to use comfortable (e.g., LibreOffice), using lighter application packages is better (Dillo, Links2, Leafpad, Geany, etc.), and CLI is better still (links2 -g, etc.). I'm a bit worried about how the CPU is often maxed out at 100%.

- If the Console Terminal screen size can be fixed to use the full 1024x768 display and the Sound system somehow enabled (it's actually quite good sound quality in DOS and Windows 98, I was just doing some retro-gaming) then this could be useful and cool mainly cli computer (I've found dozens of great cli apps in the antiX-cli-cc package manager and doing google searches, there is a bit of life with cli) with the option to do minor gui stuff. Hopefully when the cli-aptiX kernel feature is implemented the fallback kernel can handle the sound.

- Again, I offer this as a test for pushing the lower limits of what is possible (maybe not recommended) with antiX 17b1.
