A lot of the recurring trouble we have had with running persist-save at shutdown is due to trying to run it in a GUI interface while X is still running. We have to do this differently in antiX and in MX. It is running twice now because both mechanisms are kicking in. If we are willing to have it run in text mode then we don't need to play any tricks and we can make sure it runs once and only once regardless of whether X windows is running or not and regardless of how the system is shutdown (or rebooted).

So I'd like to have persist-save run at shutdown/reboot in text mode, after X closes down. Any objections or concerns?

BTW: you can fix the current problem of persist-save running twice by deleting"halt" and"reboot" in /usr/local/bin.
