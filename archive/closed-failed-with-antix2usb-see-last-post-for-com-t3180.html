Edit:
La façon simple et directe de créer une clé bootable est de copier le contenu de l'ISO:

Code: Select all

# dd if=/home/yves/Downloads/antiX-M11-686.iso of=/dev/sdX
1392640+0 enregistrements lus
1392640+0 enregistrements écrits
713031680 octets (713 MB) copiés, 271,758 s, 2,6 MB/s
Bien sûr vérifier deux fois sur quel device on copie ( /dev/sda ou /dev/sdb...)
Cela fonctionne avec antiX-M11
Depuis M11 l'ISO est un ISO hybride qui utilise isolinux comme chargeur pour démarrer aussi bien le CD que la copie de l'ISO sur périphérique USB.

hello to all

i'm finaly back to unetbootin/ext2

4 G key formated as 1,7G ext2 fos system (sdb1) and 2,OG ext2 for storage (sdb2)

installed antix M11 via unetbootin on sdb1 from a debian/sid (aptosid) box (i have the last
extlinux package installed) with 1000Mo space for persistance

edited the extlinux.conf to add lang=fr in the"default" line

rebooted
this way persistance do not works : so i re-edited the extlinux.conf (from the debian box)
on the key and modifies the default as :

code :

label unetbootindefault
menu label Default
kernel /ubnkern
append initrd=/ubninit quiet lang=fr nosplash noxorg nouveau.modeset=0 rootfs=UUID=ez77c196-158c-449b-5949-9cc3c2fc3148

removed the"persistent" from unetbootin and replaced by rootfs=uuidxxxx, where the uuid is the one of
the 2G storage partition of the key (get that with ls -l /dev/disk/by-uuid)

this sets up persistance, ok. Note the first partition could have been 1000Mo smaller as all modification will go to second partition now

so now i have a functionnal live antix on a stick, with french preselectes and persistance !

boot is fast, closing session a little slow (when saving data), do not remove the key before shutdown
completes
