A little update:
i asked on the firebox mailing list about the deskmenu and was shocked how fast they responded.
Hello,

About one week ago, I had a look at DeskMenu [1] in order to allow
such a program to work fine with Firebox.

It appeared that DeskMenu needed to be modified to run with an
improved version of Firebox. Indeed, it uses an obsolete Gnome hint
called _WIN_DESKTOP_BUTTON_PROXY [2]. If the current WM doesn't
support this hint, then the program exits.

So I asked the maintainer of DeskMenu, Stefan Pfetzing, if he would be
interested in a patch allowing not Gnome compliant WMs to use
DeskMenu. He said yes. Unfortunately, I learned a few days ago that
he's quite busy at the moment, so he doesn't plan to work on this
soon.

So you can find enclosed the patch I sent. If you try it, feel free to
give feedback !

The patch works with the SVN r242 and the new command line flag
--no-root-menu (for Firebox).

Regards,
Cyrille Bagard

PS: the subversion version currently generates lots of warnings ; the
situation is going to be fixed.
