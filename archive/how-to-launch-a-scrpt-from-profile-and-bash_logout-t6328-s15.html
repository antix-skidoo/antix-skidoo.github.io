Are you running a min-*wm option?
If you are the startup files and programs set to start in desktop-session.conf will be ignored, and the window manager startup file renamed. Min- starts only the minimal number of items that being essentially only the window manager.
If not try renaming ~ / desktop-session, then make that same directory and copy / etc / desktop-session /desktop-session.conf to it as well as / etc / desktop-session / startup and see what happens.

Also if there is a log file in ~/ .desktop-session that would be good to see.
