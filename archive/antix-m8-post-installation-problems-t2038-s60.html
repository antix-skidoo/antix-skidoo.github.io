- Some general tips first.
For you to 'find yourself' in the system regarding some application it's good to know what files are related to it. For instance if you install firehol then you open Synaptic (if it isn't opened already), search for firehol, select it, click properties and see the 'installed files' (all in French for you __{{emoticon}}__ ).

All the /usr/share/man/ (terminal man pages) and /usr/share/doc/ are documentation. For firehol there are FAQ, README, examples and a lot of other things that can be very instructive.
Usually the /bin or /sbin files are the executables and the files under /etc/ are the configuration ones.

If you install firehol then you could open Dillo and open the file /usr/share/doc/firehol/html/index.html

- The config files here are:
/etc/firehol/firehol.conf

Code: Select all

version 5
# Accept all client traffic on any interface
interface any internet
        protection strong 
        server"icmp ping ICMP ssh" accept
        client all accept
and /etc/default/firehol

Code: Select all

#To enable firehol at startup set this everything != NO
START_FIREHOL=YES
#If you want to have firehol wait for an iface to be up add it here
WAIT_FOR_IFACE=""
this one I set so it starts automatically.

Also you may open rcconf in a terminal as root or with sudo rcconf if you set sudo and then enable firehol.

Then to start it without needing to reboot you may, as root

Code: Select all

/etc/init.d/firehol start
