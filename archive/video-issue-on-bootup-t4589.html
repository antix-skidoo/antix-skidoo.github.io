I don't see an attached log file.

If you're running the LiveCD then you can specify the X resolution with the"xres=WWWxHHH" boot option. where WWW is the width and HHH is the height. You can also specify a video driver with the"xdrvr=XXX" boot option. These option won't work on a installed system but they should carry over from the Live system when you do an install. If you specify either option then we will create an xorg.conf file for you. This is what carries those options over when you do an install.

You will probably still need to select the"safe" video mode from the"F5 Video Mode" menu which adds the nomodeset boot parameter. You should certainly try the"very safe" option from that same menu (without specifying xdrvr or xres). We won't make an xorg.conf file in this case so you get narrower range of vertical and horizontal frequencies. It will be less likely to get the maximum resolution but if the WSoD is caused by a sync problem then this might avoid it.

It might be helpful if you showed us the output of"inxi -F". I'd also like to see what is in the .xsession-errors* file(s) in your home directory.

This is very strange. It seems like there are two problems but maybe they have a common cause.

When the WSoD occurs, you can try logging into a virtual terminal with <Ctrl><Alt><F2>. You can switch back to X-windows with <Ctrl><Alt><F7>. If you are getting dumped to a graphical login manager (SLiM) on the Live system then this means X and/or the window manager has crashed or failed to start. In this case it would be good to make a copy of the .xsession-errors file and the Xorg log file. OTOH it is normal to first get the graphical login screen when booting an installed system which means X is probably not crashing.

You can increase the resolution of the virtual terminal with the"vga" boot option."vga=795" will give you 1280x1024 and"vga=792" will give you 1024x768.

With a Gig of RAM you should be able to use the"toram" feature on the LiveCD which will make it run very fast. There is a delay when you boot while almost the entire LiveCD gets copied into RAM. You can either type in"toram" directly or select it from the"F4 Options" menu to make this happen.

My guess is that it is a sync problem. You may need an xorg.conf with a custom mode line. We don't create this automatically. If you want to experiment starting X in different ways, one trick is use the"3" boot parameter which will prevent us from starting X automatically. Then log into a virtual console as the demo user (or the normal non-root user on an installed system) and run:

Code: Select all

xint
which should start X-windows with the default windows manager. When you log out of the window manager then X-windows will stop.

You can move between virtual consoles with <Ctrl><Alt><Fn> where n can be 1 -- 6. You can also move to"adjacent" virtual consoles with <Alt><Left-Arrow> and <Alt><Right-Arrow>.

It's possible that just switching to a virtual console and then switching back to X-windows will fix the sync problem without you having to wait for screen-saver mode. This is not an ideal solution but it would be a significant improvement.
