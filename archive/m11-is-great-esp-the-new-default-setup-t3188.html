Kudos to anticapitalista and the development team for pre-activating the ROX pinboard desktop environment. This change broadens the appeal of antiX Linux, yet still gives longtime users who don't like desktop the environments the option of deactivating ROX pinboard. I was a Windows-only user before I started using Linux in early 2007, and I can tel you that Windows users like having a desktop environment and clickable icons on the screen.

I like the replacement of the firehol firewall with ufw. firehol in M8.5 kept giving an error message about the get-iana script's failure to update. ufw, on the other hand, works without tweaking.

Windows XP is still widely used, but support for it ends within 3 years. I see this as the biggest opportunity for Linux since Windows Vista was introduced. I'm not sure how well heavyweight distros like Ubuntu, Mint, Fedora, etc. can capitalize on this opportunity due to their high and growing hardware requirements. On the other hand, antiX Linux and other lightweight distros are substantially lighter and faster than Windows XP.

The upper part of the Linux market is very competitive but the lower part of the market is sparsely populated. The top then distros listed on Distrowatch consist of 7 heavyweight mainstream distros, Debian, Arch, and only one lightweight mainstream distro (Puppy). There aren't that many other lightweight mainstream distros - antiX Linux, Swift Linux, and Lubuntu. (I question Lubuntu's ability to remain lightweight in the future given that it's based on Ubuntu. Ubuntu has its merits, but lightweight operation isn't one of them.)

I see antiX Linux as the Ubuntu of the lower part of the Linux market (and my antiX derivative Swift Linux as the Mint of the lower part of the Linux market). antiX has a large repository (unlike Puppy) but is as easy on RAM as Puppy Linux and almost as user-friendly.

The main challenge I see for antiX Linux is keeping the full version small enough to fit onto a CD. The full version is already up to 680 MB. Given that all distros (even Puppy Linux) have a trend towards bigger ISO files, I'm expecting the challenge to grow further over time. In the development of Swift Linux, I have to cut packages in order to make room for pre-installed OpenOffice.
