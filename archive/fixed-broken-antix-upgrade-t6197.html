An upgrade will try to install a newer version of antix control centre which includes a new deb ds-mouse-antix and removes mouse-cc-antix.
It complains of the desktop file not being removed, but the install gives errors.

Fix for now.

If you have done the upgrade;

Code: Select all

dpkg --force-overwrite --purge mouse-cc-antix antix-cc-full-antix (or antix-cc-base-antix if using base version of antiX)

apt-get install antix-cc-full-antix
If you have not done the upgrade, before doing so

Code: Select all

apt-get purge mouse-cc-antix
apt-get dist-upgrade
A fixed deb will appear later. (I'm out all day and evening today)
