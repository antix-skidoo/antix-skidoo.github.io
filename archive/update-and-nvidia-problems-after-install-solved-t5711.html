    For 2. Try using another mirror to see if the problem persists. Change es to de for example in etc/apt/sources.list.d/debian.list

    For 3: It can be done, at least with the nvidia card I have. But it is not easy.

    1. make sure the fixes repo in siduction.list is enabled
    2. when you boot, best to do so to init 3 and with these cheats nomodeset nouveau.modeset=0
    This will make sure you do not load the nouveau driver.
    3. login as root, type smxi and at the graphics section, choose the option debian way.

    That is how I managed it.

    Aleph wrote:...W: Fallo al obtener http://ftp.de.debian.org/dists/jessie/updates/main/binary-amd64/Packages 404 Not Found
    It looks like you've changed the base url of the Debian security repo from"http://security.debian.org/", to"http://ftp.de.debian.org/". {the security repo is the one that has"jessie/updates" in it's repo entry}

    The Debian security repo is always at"security.debian.org", so you should change the repo entry back to that.

    Here's the original /etc/apt/sources.list.d/debian.list for reference, except it's set to the US mirror.

    Code: Select all

    # Note:If you want maximum stability, keep to the default Stable/Jessie repos.
# Jessie is the new stable release and will only recieve security 
# updates, while Testing will be 'rolling'.

# Debian Stable. Default for antiX-15.
# Use for maximum stability INSTEAD of the 'rolling' TESTING release concept.
deb http://ftp.us.debian.org/debian/ jessie main contrib non-free 
deb http://security.debian.org/ jessie/updates main contrib non-free
#deb-src http://ftp.us.debian.org/debian/ jessie main contrib non-free
