I would avoid ubuntu packages in general in antiX/debian. there are significant library differences in the two os's, as well as directory structure differences. You may get lucky occasionally, but eventually something will break.

check out the meta-packageinstaller in the control center>system tab, as well as synaptic for general installations.

simplescreenrecorder is not available in the default repos at the moment, but vokoscreen and gtk-recordmydesktop are available.

you could enable the mx15 main repo in the synaptic options if you like. simplescreenrecorder is available there, and the packages *should* all work with antiX15. just disable it when you are done installing what you want.

i would like to install new applications, (wine, playonlinux, winFF, pcsxr, qbittorrent, pingus, skype, teamviewer, okular, suite educativa gcompiz, tux paint, shutter, acetoneISO, simple screen recorder, etc... )
Without intending to sound"harsh", I'll suggest that antix15 probably is not a good distro choice for you.
First of all, choosing a distro with wine preinstalled is probably advisable, b/c installing and setting up wine is PITA.
Second, because all the"stuffs" you intend to install isn't common to other users of this distro,"who ya gonna call?" when you need help with those applications?

Installing Okular will pull in a shitboatload of KDE dependencies. qBittorrent brings a raft of qt framework dependencies. Shutter brings gnome/gvfs dependencies...
The result will be a frankenstein, difficult to troubleshoot, and far-removed from the antix starting point.
