1. It shouldn't be obligatory. On my box I can use nouveau, and vesa (as well as nvidia).

2. nouveau.modeset=0 is to stop nouveau drivers from loading. Needed if you use anything bar nouveau and do not have an /etc/X11/xorg.conf. Default installed antiX has no xorg.conf and defaults to vesa.

First stage of testing.
This is with latest, unreleased antiX-base(modified), using default repos.

For nouveau, no xorg.conf is needed and there should be no reference to nouveau in the boot/menu.lst. Like this with default MEPIS kernel

Code: Select all

title antiX-test (nouveau)
root (hd1,1)
kernel /boot/vmlinuz-2.6.36-1-mepis-smp root=/dev/sdb2 nomce quiet vga=791 
initrd /boot/initrd.img-2.6.36-1-mepis-smp
I then installed latest liquorix kernel and headers via repo, not using smxi. This works fine.

Code: Select all

title antiX-test (liquorix-nouveau)
root (hd1,1)
kernel /boot/vmlinuz-3.0.0-4.dmz.2-liquorix-686 root=/dev/sdb2 nomce quiet vga=791
initrd /boot/initrd.img-3.0.0-4.dmz.2-liquorix-686
I then tried liquorix kernel with vesa driver. I needed this .

Code: Select all

title antiX-test (liquorix-vesa)
root (hd1,1)
kernel /boot/vmlinuz-3.0.0-4.dmz.2-liquorix-686 root=/dev/sdb2 nomce quiet vga=791 nouveau.modeset=0 nomodeset 
initrd /boot/initrd.img-3.0.0-4.dmz.2-liquorix-686
If you are certain you do not want/need nouveau drivers (or any other xorg modules) run smxi -G as root in X and choose 3. clean-up stuff then 7. remove xorg-modules. I removed all suggested except fbdev.
