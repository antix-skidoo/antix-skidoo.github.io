slim is messed up, either purge it and reinstall it as root or check the settings for it.

Code: Select all

apt-get update
apt-get purge slim
apt-get install slim
the settings are in / etc/slim.conf and you should have a sample config in / usr/share/slim/slim.template

Quite frankly, if you set the Debian software archives to the Debian Testing packages, you can, for the most part, keep the same release, regardless of what it is, and perform periodic, perpetual updates or upgrades. That said, any time there are major Debian infrastructure changes, you can't just blindly upgrade and expect perfect results without any thought or consideration. But if you use the smxi system administration tool as your mechanism, instead of apt-get, aptitude, or synaptic, the maintainers and the community of smxi keep a pretty close eye on the changes taking place in the Debian software archives that it supports.
