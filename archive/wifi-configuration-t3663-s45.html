I think you are on the right track, Roky. However, if the correct firmware is installed and if this device physically works elsewhere on other operating systems, what that tells me is that the firmware is being blocked from being properly executed.

If modprobe atmel_cs

gets it done, great, nearly there (just may have to repeat that). But if it doesn't, what that tells me is that there is ANOTHER firmware driver out there somewhere that is conflicting with the atmel_cs firmware. In that case, more research is needed. What firmware is conflicting? Once we know that, we can put that firmware into a blacklist, then at last we'll get the device enabled. Then, if it still doesn't come up automatically, we can explicitly prove for it in a modprobe.conf file and be done with this.

So the next challenge is to determine whether the probe works or not, and if not, why not. I think we can get this, ESPECIALLY if it's working on Ubuntu and other systems. Very worst case is that if it works on Windows, we can snag its firmware, blacklist any other similar firmware, and use ndiswrapper to get this going. It can definitely be done; ndiswrapper, in my opinion, is a last resort. The superior solution, and the one that will perform the best, is getting rid of conflicting firmware and/or blacklisting it.

Before trying what I am going to suggest here. I wanted to make sure you did not have atmel_cs blacklisted first. That is what my post above covers. If atmel_cs is blackisted. It should show in the above commands I posted.

Ok. Lets say atmel_cs does not show as blacklisted.

Open rox as root in terminal and go to /etc/modprobe.d and open the file.
Make a new text file.
Name it something like blacklist-custom.conf

inside type in

Code: Select all

# /etc/modprobe.d/blacklist-custom.conf
# Custom blacklist file so I don't mess with any of the files that come with
# the module-init-tools package.
blacklist at76c502
save it. Reboot. See if atmel_cs takes over as driver for the Xircom card then. This is all experimental to me
as I have never had to blacklist a module from loading in AntiX ever since I have ran it since AntiX 7. So if I am off base here. I hope other forum members will correct my post/me. __{{emoticon}}__
