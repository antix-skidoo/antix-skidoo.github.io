Oops! My solely fault!

You can see the error copying the menu entry of Mepis 14 to AntiX 15 in the original post.

So, the elaborated menu entry of grub.cfg in AntiX 15:

Code: Select all

menuentry 'antiX GNU/Linux' --class antix --class gnu-linux --class gnu --class os $menuentry_id_option 'gnulinux-simple-5949696b-1554-41d1-8ae7-71b0385d9a70' {
load_video
insmod gzio
if [ x$grub_platform = xxen ]; then insmod xzio; insmod lzopio; fi
insmod part_msdos
insmod ext2
set root='hd0,msdos12'
if [ x$feature_platform_search_hint = xy ]; then
search --no-floppy --fs-uuid --set=root --hint-ieee1275='ieee1275//disk@0,msdos12' --hint-bios=hd0,msdos12 --hint-efi=hd0,msdos12 --hint-baremetal=ahci0,msdos12 [i]UUID[/i]
else
search --no-floppy --fs-uuid --set=root [i]UUID[/i]
echo 'Loading Linux 4.0.0-antix.1-486-smp ...'
linux /boot/vmlinuz-4.0.0-antix.1-486-smp root=UUID=[i]nº[/i] ro quiet
echo 'Loading initial ramdisk ...'
initrd /boot/initrd.img-4.0.0-antix.1-486-smp
}
Can be"resumed" in burg.cfg like:

Code: Select all

menuentry 'antiX GNU/Linux' --class antix --class gnu-linux --class gnu --class os {
insmod ext2
set root='(hd0,12)'
search --no-floppy --fs-uuid --set  UUID 
linux /boot/vmlinuz-4.0.0-antix.1-486-smp root=UUID= [i]nº[/i]
initrd /boot/initrd.img-4.0.0-antix.1-486-smp
}
