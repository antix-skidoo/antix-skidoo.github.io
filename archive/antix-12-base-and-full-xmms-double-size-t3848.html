It's amazing what happens when you walk away from a problem for a few hours. I've worked out a way of doing both the export and starting XMMS to use double size without modifying the antiX menu or WM's.

Create the script /usr/local/bin/xmms

Code: Select all

#!/bin/sh

# Tweak to enable XMMS to use double size mode
# This script must be stored in a directory that is listed in $PATH ahead of the directory containing the xmms binary
export XLIB_SKIP_ARGB_VISUALS=1
/usr/bin/xmms"$@"
Set the ownership to root:root
Set the permissions to be rwxr-xr-x

Finished.
It will now start from the usual (unmodified) antiX menu entry and work in double size mode.

The reason it works is that both script and binary have the same name. The script is located in a directory that is searched before the directory holding the binary. The antiX-12 menu entry refers to xmms without specifying a path and therefore finds the script before the binary. The script conducts both the export and starting the binary in the same environment, which in turn allows the use of double size mode.
