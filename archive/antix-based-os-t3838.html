IIUC, you are more than welcome to make a custom OS based on antiX. Almost all the code in antiX is either Free Software (GPL) or open-source (more permissive BSD type license) which means you don't need to be given permission to make a custom OS. That permission is already granted in the licenses.

In fact, we have gone to great effort to make it very easy for you to make a custom OS. For example, the remaster-me feature in the LiveUSB allows you to make a custom version with just a few mouse clicks in a GUI. Simply make the modifications you want to a LiveUSB and then use the remaster feature.

There is one caveat however. A lot of the software in antiX including the Linux kernel and most of the antiX specific software is licensed under the GPL. The GPL gives you the right to modify the software and make your own versions which you can either give away or sell. The one stipulation is that if you modify the software and then give it away or sell it then you must also make your source-code modifications available to whoever you give or sell it to.

All this GPL software is offered freely to the world as a gift. The only catch is you are not allowed to distribute derivatives without also distributing the source code. It is a safe-haven for people who want to ensure their software gifts to the world remain Free (as in freedom, not free as in beer. You are always free to sell GPL software).

If you are interested in creating closed-source derivatives then you need to start out with code that uses a more permissive license. The BSD projects all use such licenses. Apple's OS X is an example of a closed source BSD derivative. Apple used the freely available open-source BSD code to create their own OS but then made their OS closed-source and they refuse to share the changes they made even with the people they got the original BSD code from. This is their right according to the BSD license.
