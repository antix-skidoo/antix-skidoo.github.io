Eino wrote:Simple upgrade from alpha2 to beta1 via smxi that's all.
yes, that will work; you will have all available packages. What you will not automatically get are packages that are new, changed, or removed; an smxi change simply manages the repos that you have set up. So you will have responsibility, if you use that means, to change what you want; what's changed IS available (always) in the repos that are included in the distro.

This rfkill issue isn't a problem - it is expected behavior, and not to do with drivers. And yes, there are many people out there to whom it will never happens to either.

As I understand it, rfkill is used to switch off your wireless chipset to save power - useful for folk with laptops etc. And if rfkill defaults to unblocked, and you never knowingly block it, then rfkill issues will never arises.

But what if someone runs windows, and they click the checkbox to disable wifi. or similarly with Network Manager, there is a checkbox there that does the same. Such that if someone has been in another OS, left rfkill blocked, and then booted into AntiX, then their wifi is left basically unusable and unfixable.

you should install live-init-antix_0.4.5_all

Make sure that the sources list included dev in antiX repo and do not include full (or base or core)

This is how it should be for running bleeding edge antix development.

Code: Select all

# antiX
deb http://www.daveserver.info/antiX/debs testing main dev
At the moment, everytime a new desktops-default file gets uploaded, it changes the repo to remove dev. This is for safety reasons at the moment so an upgrade doesn't destroy installs.
