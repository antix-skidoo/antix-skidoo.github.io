You can install the downloaded debfile using the command dpkg -i /path/to/rox-filer_2.12-ski_amd64.deb

This new version will not disturb your existing rox-filer configuration files.
It will write, and read from, files named"Options3" and"menus3" (ignoring the existing files, named"Options" and"menus2")
This new version has been successfully tested in antix16 and MX16 (sans"desktop-defaults-rox-antix")

shortlist of notable changes:
  • in details view, columns are reorderable
    (non-permanent ~~ custom order is discarded when rox-filer exits)
    .
  • toolbar: additional buttons are available, and you can choose which buttons to show
    .
  • optionally, toolbar buttons may be set to"display textlabels only (no icons)"
    .
  • Double click on free space of the views.
    If view is already auto-resized, window will expand to full size for sake of file drag/drop convenience
    .
  • option to choose thumbnails size (small 64px, normal 128px, large 256px and huge 512px)
    .
  • improved window auto-resize behavior (when items added, renamed, removed, or item updated)
    .
  • Bookmarks: changed to save basename as a title (Full path was not easily viewable)
    .
  • fixed ownership preservation while copying/moving dirs and symlinks (as root)
    .
  • Regex Select" - just type some characters and all items containing that phrase will be highlighted
    ( try: type"local" in /usr/bin ) (also, refer to the screenshot attached to this forum post)
    .
  • you can save custom window position & size, display style (and more) for individual directories
    (if a custom icon has been assigned to a dir, that icon will display in that window titlebar while browsing the dir)
    In"icon view", thumbnail images are shown for directory items with custom-assigned icons (vs stock 'dir' icon)
    .
  • option to force zebra-striping of alternate lines in details view, in case your preferred GTK theme lacks striping
    .
  • Pressing ESC ESC clears the multiple selection; pressing ESC '!' sends your selection straight to the command line.
    .
  • new, optional (must enable via Options dialog) LINKED WINDOWS functionality.
    Provides side-by-side, locked-together windows ~~ ala fluxbox"tabbed windows".
    (avoids the need to Alt+tab in order to keep track of multiple opened, stacked windows.)
    -=-
    If enabled, can be activated by selecting a directory item in the rox-filer view, then performing
    mouse scroll while hovering an empty spot on rox-filer toolbar.
    While LinkedWindows mode is active, special glyphs are shown in the titlebar of each of the dual windows.
    You can independently navigate within each window, with quick drag-n-drop access to the sibling window.
    .
  • upon navigating to parent directory, rox provides a visual callout (wink+focus) indicating the subdir you just exited
    .
  • provides visual differentiation: pointing hand cursor when hovering clickable item
    (vs normal cursor when cursor is pointed at background, aka"blank space" aka"space between items")
    This is important usability consideration b/c right-click menu differs based on the context
    .
  • right-click of toolbar"Up" button will open a new window containing the PARENT of the currently-viewed dir
    .
  • clicking column headers performs columnsort view (pre-existing feature?)
    .
  • All references to"0install","zeroinstall","0launch","0compile" have been removed from Rox-Filer Manual.
    Functionality related to these mechanisms is now deprecated, and the supporting code has been removed.
    .
  • revised and expanded labeltexts and tooltips throughout the Options dialog tabs
