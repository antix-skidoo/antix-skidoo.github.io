firefox replaces iceweasel. You can remove the iceweasel desktop icon.

Howto change toolbar background color? Depends on which window manager you are using ~~ which is a detail you didn't mention. Are you saying the color does not change when you select various different window manager styles/themes?

Can I put a shutdown button plus other option buttons on the toolbar as in 13.2 ?
Again,"howto" depends on which window manager you are using ~~ which is a detail you didn't mention.

Why is the install button left on the menu when the install is complete ?
Idunno. Maybe you performed a snapshot, and the snapshot utility re-created a launcher entry?
This will remove the launcher file:
[color-green]sudo rm /usr/share/applications/antix/minstall.desktop[/color]
You can remove the menu entry by visiting ControlCenter--} Edit JWM Settings
(or fluxbox or icewm) and search for, and delete, the line mentioning the"install" menu entry.
Followup by clicking"update Menus" within the desktop menu.

Where can I find the dialogue to define preferred applications in a list when a file name is clicked ?
Sadly, there apparently is no"one single place" to do so, because rox-filer file manager and spacefm file manager differ in how they"pick" handlers for various file types. Your starting point would be to use the"Preferred Applications" utility available in antix ControlCenter. Afterward, you will probably ALSO need to tweak the options/preferences settings within the spacefm and/or rox-filer dialogs.

"I was using the app to write an iso file from a bootable CD. Is there an alternative ?"
within the desktop menu:
Applications --} System Tools --} iso-snapshot
Perhaps this is what you want?

Bleachbit was probably omitted in the newer build due to trying to minimize the ISO size.
So just sudo apt update && apt-get install bleachbit

"13.2 takes up to 5 minutes for the same. Any reason for such a difference ?"
If it's working well, instead of guessing"Why?" let's just be thankful.
At times I can hear the hard disk working away
for a short while
for no apparent reason
when logged into email, various forums or web browsing under Chromium or Firefox.
The newer web browser versions do consume more RAM compared to older versions. Can we rule out"swap access" in the scenario you've described?
If swap is not being accessed, you can alleviate firefox disk I/O by visiting about:preferences and editing the prefkey browser.cache.disk.parent_directory
(or create a key with that name, if it doesn't already exist)
and supply a tmpfs path for the prefkey value, e.g. /tmp/happyfox

The bleachbit password prompt behavior you described is the expected behavior.
Okay, _you_ didn't expect it... I'm just explaining it's not a bug.
You'll find the same (wants YOUR password) when launching sudo-permissioned ControlCenter apps, or Synaptic Package Manager, etc
installing the group of Terminal Apps as a single update or do I have to go through them one by one and set up the menu entries manually ? Can I set them all up as an executable file and run them under bash?
What is"Terminal App"?
You might be referring to custom shell scripts, except... you're asking how to"install" them? Confused.
-=-
"Can I set them all up as an executable file and run them under bash?"
Yes.
-=-
To gain ability to launch via menu, create a .desktop file for each item you want added to the menu, then perform"Update Menus"
What browser do you suggest I use ?
I won't"suggest". It's a personal decision -- you can try various browsers to decide what best suits YOU.
Maybe some sites you visit (geemail) behave better when using"gbrowser", yet you'll prefer fbrowser (or pbrowser, or qbrowser) for general use?

"SQL database"
I don't use chrome/chromium, but it's probably storing data in a"sqlite" container.
Editors are available for inspecting/editing sqlite datafiles (no, I don't have specific download links).
IIRC, debian has a"sqliteman" package; I can't recall whether it provides a GUI.
If you do decide to edit, create a backup copy of the datafile.
Also, in case it's not obvious... don't attempt to edit browser-related files while the browser is running.
