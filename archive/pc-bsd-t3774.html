cpulimit is a simple program which attempts to limit the cpu usage of a process (expressed in percentage, not in cpu time). This is useful to control batch jobs, when you don't want them to eat too much cpu. It does not act on the nice value or other scheduling priority stuff, but on the real cpu usage. Also, it is able to adapt itself to the overall system load, dynamically and quickly.
The following works well here (on Gentoo):

Code: Select all

# burnK7 &
# cpulimit -e burnK7 -l 50
Process 9558 detected
[1]  + suspended (signal)  burnK7
I stopped it with ctrl-C then"fg" command then another ctrl-C. This may not be the optimal way to control it. The cpulimit program is very tiny. It should be run as root. The burnK7 is supposed to use 100% of one AMD cpu. There are other burn programs for Intel chips.
