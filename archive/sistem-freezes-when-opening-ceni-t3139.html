Kinda off topic. But I am posting on my 2 gig Kingston antix2usb persistent AntiX 11 I686.base on my Acer Aspire One ZG5 (160gig sata drive version 1 gig of ram Atom processor)

This is a different breed from a full install of base to hardrive. Ceni connects OK via hardline but wlan0 in ceni for me is always a no go (inept carbon face interface probably) __{{emoticon}}__ .
Rutilt segfaults from command line. So to get wireless going I connected to landline via ceni first. Then

Code: Select all

sux
type in"root" for password.

Code: Select all

apt-get update
let it finsh

Code: Select all

apt-cache policy wicd
When version of wicd shows. I go ahead and install wicd.

Code: Select all

apt-get install wicd
When done install wicd. I do

Code: Select all

wicd &
To make sure the wicd deamon started up. Then to run wicd as user

Code: Select all

exit
Hit enter
Then type in

Code: Select all

demo@antiX1:/aufs/home/demo$ which wicd-gtk
/usr/bin/wicd-gtk
to make sure where the gui command is and that it is there.

Then type in

Code: Select all

wicd-gtk
Click on the icon in my slit in fluxbox. Add wlan0 to preferences in wicd under wireless interface.

Go ahead and hit refresh and connect up with wicd. Open up ceni and remove my eth* connection config interface I made earlier. Because from now on I will be using wicd for hardline also.

Code: Select all

ifconfig -a
as root in terminal will show what connections are available for wicd when i change hardware when I plug this pendrive into another computer. Whether wireless or landline also.

Last I go into antix control center and edit fluxbox settings/startup tab in geany.

Code: Select all

# wmnd &
# wmsmixer -w &
 wicd-client &
# halevt &
Just uncomment/remove # in front of wicd-client & to autostart wicd during bootup. Preferences like dhclient instead of Automatic or measure signal strength in db is up to you and your hardware. Whatever works for you to connect.
