In M-11, Iceape provided Iceape Composer, a really simple html editor that was ideal for casual production of mainly text based documents. Installing part of the Iceape suite (iceape-browser) also installs Iceape Composer. It seems that this also changes the default browser recognized by antiX12.

In the Iceweasel interface, Preferences-->Advanced-->System Defaults-->Check Iceweasel is Default indicates it is default, but it is not responding as the default.

With no browser running, opening a web address via right-click-->Open URL in Browser opens it in Iceape rather than Iceweasel. How does one reset Iceweasel to be used as the default?

You should also be able to edit /usr/share/applications/mimeinfo.cache for global defaults and ~/.local/share/applications/mimeapps.list mimeinfo.cache for user defaults. The first app under the file type you are tring to open is the default application.

Example in your case:
text/html=iceweasel.desktop;iceape-browser.desktop;chromium.desktop;elinks.desktop;geany.desktop;

Note: though your global default may be correct, your user default overrides it for your user.
