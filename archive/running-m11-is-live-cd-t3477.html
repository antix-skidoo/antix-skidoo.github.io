vinoman wrote:It does boot up, it doesn't let me go in to live cd mode, it wants me to log in as root and do a cli-installer.
It sounds like you are using an antiX-core LiveCD. When this boots up you get the following message on the screen:
For cli install, login as root (password=root) and type cli-installer.
These are merely instruction on how to install antiX from the command line. You are free to ignore them. It is a LiveCD but it only has a Command Line Interface. There is no Graphical User Interface (GUI) on the antiX-core LiveCD.

If you want a GUI then you need to download and burn either an antiX-full iso or an antiX-base iso. Here are 3 recent antiX iso's:

Code: Select all

$ ls -lh
-rw-r--r-- 1 BitJam BitJam 680M May  2  2011 antiX-M11-686.iso
-rw-r--r-- 1 BitJam BitJam 364M May  2  2011 antiX-M11-base-686.iso
-rw-r--r-- 1 BitJam BitJam 115M May 28  2011 antiX-M11-core-squeeze-486.iso
The first one, that takes up 680 Meg, is the full distribution. It has all the bells and whistles.

The second one, that takes up 364 Meg is the base distribution. It is a little more than half the size of the full distro. A lot of stuff was stripped out but it still has X-windows and a Graphical User Interface.

The third one, with only 115 Meg, is the base CORE (edit by rokytnji) distribution. This is the one you have. It is very small because it doesn't even have X-windows and thus it doesn't have a Graphical User Interface. You can log in as root (password: root) and you will have a Command Line Interface (CLI).

If you want a Graphical User Interface (using a mouse and so forth) then you need to download either the base iso or the full iso and use that to create a new LiveCD.
