ah, an important clue! My vbox setup has a"vboxvideo driver". make a file in /etc/X11/xorg.conf.d . I called mine 10-monitor.conf. You have to have a two digit number and the dash in front, from what I've read. cut and paste the following code into it and save. reboot and see what you get. I got a host of new resolution choices, but most were too big for my little monitor. but xandr reported 1280x1024 resolution and 24 bit color depth.


/etc/X11/xorg.conf.d/10-monitor.conf

Code: Select all

Section"Device"
    Identifier   "Configured Video Device"
    Driver       "vboxvideo"
EndSection

Section"Monitor"
    Identifier   "Configured Monitor"
    Option       "DPMS"
EndSection

Section"Screen"
    Identifier   "Default Screen"
    Monitor       "Configured Monitor"
    Device       "Configured Video Device"
    DefaultDepth    24
    SubSection   "Display"
        Depth    24
        Modes   "1280x1024"
    EndSubSection
EndSection

Section"ServerLayout"
    Identifier   "Default Layout"
    Screen       "Default Screen"
EndSection

secipolla is correct. We all should have more or less the same VB"hardware" as its a total virtual machine. All the guest OS can see is the virtual machine, not the actual hardware. Even the ethernet connection is virtual. On a windows host machine, an extra network driver/connection is used to bridge the virtual/guest ethernet to the actual/host network connection. Even if you are using wifi on the host, the guest will always show a hardwired connection.

I played around with a second host/guest setup last nite. A couple of notes: whatever resolution you put in the .conf file will be what X trys to use. Other options may be present in xrandr, but X will default to the one specified in the .conf file. A resolution 1 step down from your systems native resolution is probably a practical max because the VBOX window will take up some space on the host screen. and a typo in this file will probably cause X to crash out (found that out the hard way __{{emoticon}}__ )
