As a followup to what thriftee mentioned about border width,
and toward facilitating self-help in globally overriding the width of iceWM themes
I'm suggesting to add the following into the foot of /etc/skel/.icewm/preferences

@thriftee
for immediate use, you can visit
ControlCenter }} Desktop }} Edit iceWM Settings
paste this into your preferences file, and begin experimenting with setting global border (and resize corner) widths

Code: Select all

#################################
###
### TO SEE OTHER AVAILABLE iceWM CONFIGURATION PARAMETERS,
### VIEW /usr/share/icewm/preferences IN A TEXT EDITOR
### AND COPY/PASTE ANY LINES OF INTEREST TO YOUR ~/.icewm/preferences
### 
### AFTER YOU REMOVE THE LEADING (#) FROM A LINE AND TWEAK THE PARAMETER VALUE
### AND Save THE EDITED preferences FILE, 
### RESTART iceWM TO SEE THE EFFECT OF YOUR EDIT
#################################
###    THE FOLLOWING ARE (OUT)COMMENTED BY DEFAULT.
###    YOU CAN GLOBALLY SPECIFY/ENFORCE WIDTH OF iceWM WINDOW BORDERS,
###    THEREBY OVERRIDING BORDER SPECS STATED WITHIN INDIVIDUAL iceWM THEMES 
###    BY UNCOMMENTING

###  Consider border frames when maximizing horizontally
# ConsiderHBorder=0 # 0/1

###  Consider border frames when maximizing vertically
# ConsiderVBorder=0 # 0/1

###  Horizontal window border
# BorderSizeX=6 # [0-128]

###  Vertical window border
# BorderSizeY=6 # [0-128]

###  Horizontal dialog window border
# DlgBorderSizeX=2 # [0-128]

###  Vertical dialog window border
# DlgBorderSizeY=2 # [0-128]

###  Resize corner width
# CornerSizeX=24 # [0-64]

###  Resize corner height
# CornerSizeY=24 # [0-64]
For JWM, I found no means of globally declaring a window borderwidth pref. User would need to edit
~/.jwm/theme
~/.jwm/theme.dark
~/.jwm/theme.light
and / or
~/.jwm/themes/*
find within the theme file, and edit:
<width>integer value</width>


For fluxbox, a global override can be declared within ~/.fluxbox/overlay
The relevant lines to edit are
window.borderWidth: integer value
window.handleWidth: integer value
window.frameWidth: integer value

@Shay
"Conky is delayed about 2 seconds"
Yes, the delay is intentional/necessary.
Within the session startup file,"sleep xx" commands are interspersed between each startup item

@dolphin_oracle
"blurred conky problem happens without changing desktops. All I did was change icewm themes."
Yes, and... the act of changing the theme in iceWM triggers a session restart.

Fluxbox has a separate"Reconfigure" command to perform same without restarting the session.
(FWIW, this internal command can be called via a keybind, and is silently called when a themename is clicked in the menu)

For testing, if you want another example item to work with, try installing"pluma" editor.
Create an autostart"pluma /path/to/some/file &"
Afterward, each time you change theme in iceWM (or otherwise trigger session restart, e.g. via"Other Desktops")
you'll see pluma issue a red warning banner stating"a copy of the document is already loaded"
