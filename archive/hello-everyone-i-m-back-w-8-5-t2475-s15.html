Ok, the multimedia repos are down.

If you want to upgrade, this is the best way to do so and you'll be 100% up to date.

1. Make a backup of /etc/X11/xorg.conf
2. apt-get purge xorg followed by apt-get install xorg
3. If you have problems with X as etc/X11/xorg.conf got removed, just restore that backup file.

As anticapitalista said, you may comment out the debian-multimedia entry in sources.list. However I have a antiX with Debian unstable repos + debian-multimedia unstable repos, a antiX with Debian testing repos (as you) + debian-multimedia testing repos and a Debian sid (unstable) + debian-multimedia testing repos and all work very well. The person that maintains the debian-multimedia repos does an excellent job and it seems to me to be perfectly compatible with Debian.

Back to the update, you now don't have Wicd anymore. In case you need it, you may follow the tip at
========= SCRAPER REMOVED AN EMBEDDED LINK HERE ===========
url was:"http://www.mepislovers.org/forums/showpost.php?p=233247&postcount=14"
linktext was:"http://www.mepislovers.org/forums/showp ... stcount=14"

to add a user called demo (I'm not sure but I think the command for that would be 'adduser --no-create-home demo') and then install wicd again.

You can notice that there are packages that are not needed anymore, these can be removed with 'apt-get autoremove'. As anticapitalista said, you may try the command smxi for the update, setting it to use apt-get dist-upgrade for the upgrade. In its post-upgrade options you will find under '3. Clean-up stuff' the autoclean (#2) as well as a clean cruft option (#8) which purges configuration files from the autoremoved apps (smxi doesn't offer the autoremove option but you can run it before running smxi).
smxi is a very useful, almost necessary, tool for Debian unstable repos since it alerts for any possible breakage of the system that has already been reported. This is much rarer with Debian testing but smxi has some other interesting things and it's good to familiarize with it.

Re. debdelta, you would have to install it (apt-get install debdelta). But don't bother with that for now. First do as anti said (purge and reinstall xorg) and update your system. Then maybe in a week or so, after you've done some updates, you install and test debdelta (but it will work only partially because of localepurge).
