In your fresh install, if you have edited the default menu, possibly the launchstring for the"Update Menus" command got mangled.
You can test by calling the operation from a terminal commandline: antixcc.sh
If this works, the content of the (~/.JWM/menu (or iceWM or fluxbox) menu file is suspect.

In your fresh install, were any of the"desktop-defaults*" and"antix*" packages marked as upgradable?
If so, have you yet upgraded those available packages?
If not, perform the package(s) upgrades, then retest to check whether the problem is resolved

hello poorguy with sigh
similarity; machine here antix16 on stretch repositories:
as soon as was seen during upgrade a message"antix=control=centre held back" I thought of Your post. Machine here is repaired, with ControlCentre invoking properly, so it does not leave any room for experimenting with various methods; I can merely relay the process undertaken that returned the Control Centre to visible working condition on one machine here:
[Control Centre was working from menu]
open terminal

Code: Select all

sudo apt-get update

Code: Select all

sudo apt-get upgrade
[got the message that package antix-control-center held back; all other upgrades succeeded. tried to open ControlCentre but it did not appear]
exit=shutdown and power on
[attempt to open ControlCentre but it did not appear]
open root terminal

Code: Select all

apt update

Code: Select all

apt upgrade
[one package antix-control-center installed successfuly. attempt to open ControlCentre and it appeared normally]
exit=shutdown and power on
ControlCentre and all working as usual. another apt update returned All packages are up to date.
We hope that Today it is easy for You like it was for here,, just another upgrade. Thank You for posting because it put us on the aware for this possibility of disappearing Control Centre. all is well. sigh.
