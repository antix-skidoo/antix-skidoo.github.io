Another issue is that the fbdev graphics driver now takes precedence over the vesa driver if the framebuffer is enabled. On the live system we enable the framebuffer by default at a resolution of 800x600 and this resolution gets carried over to X.

This only affects systems where a modeset driver is not available or when"F5 Video Mode" -->"safe" has been selected. A partial solution would be to eliminate the"safe" option and provide just"default" and"vesa". This still leaves people who have a graphics card that does not have a modeset driver with 800x600 resolution in X (such as in Virtual Box). If we can detect this situation then we can create an xorg.conf file (or a file in xorg.conf.d/) that forces the use of the vesa driver.

Ironically, even though there is a / etc/modprobe.d/fbdev-blacklist.conf file, I don't know how to blacklist the fbdev driver. It's not a kernel module and it does not show up in the output of lsmod.
