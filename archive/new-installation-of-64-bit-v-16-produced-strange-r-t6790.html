If your sda device has no remaining unallocated space, you'll need to resize the existing partitions ~~ use"gparted" and make sda1 a bit smaller, then you'll be able to make your sda2 swap partition a bit larger. This resizing can probably be performed as a post-install operation (IOW, regardless whether docs state resizing should be performed in advance, I've repeatedly done it post-install without breaking an O/S)

Regarding"installer sez you can use grub or not", a bootloader (grub, lilo, syslinux, etc) is required. The installer provides an opt-out in case you want/need to manually reconfigure your existing bootloader. So, the only reason to opt-out is if a pre-existing bootloader does, in fact, exist and you don't want the installer to overwrite it (you're planning to manually reconfigure the existing bootloader).

foggy memory. May need to right-click on the partition which is currently acting as swap and untick 'swap' flag, reboot...
