    I don't speak German. But I can use Google translate.

    If the cd device is supposed to boot first and it doesn't then maybe there is something wrong the live-cd. Do you have some other live-cd that does boot on this machine? If so then I would try to make sure the live-cd is correct. You should be able to mount the live-cd in Windows (I would hope). If it contains a single file that ends in .iso then it won't be able to boot. You don't want to copy the .iso file into a cd, you want to use the .iso file as an image to make the live-cd. That is the only way it will boot. The options to do this differ depending on the software you use to make the live-cd from the iso file.

    So to summarize:

    1) Can you boot any other live-cd on this machine?

    2) Try to mount the live-cd you made and see if it is correct. A correct live-cd should have a set of files and folders like this:

    Code: Select all

    EFI/
antiX/
boot/
cdrom.ico
version
    The trailing / indicates a folder instead of a file.

    Then the problem is very likely in the BIOS settings on the Windows-7 machine. Male provides a possible fix that involves making a live-usb.

    If you can boot a recent antiX live-cd on your other machine then you can easily convert it to a live-usb with the live-usb-maker program. Just plug in a usb stick that you are willing to over-write and run live-usb-maker either from the command line or from one of the application menus. Follow the simple directions. This is easier, safer, and better than using the"dd" command. It is a new feature that may not have been widely advertised.

    As male said, make a live-usb, plug it into the Windows-7 machine, and then use either the BIOS settings or the Boot Menu to boot from the live-usb.
