ok, i used a basic antix 11 install and ONLY updated wicd. I now get the behavior you are describing. modify the"wicd-client" line in your startup file to"wicd-client -t". This will start the wicd-tray icon as a seperate process from the main client, which apparently does not happen if you simply run wicd-client. New for version 1.7+ds9 I guess. clicking the icon will still bring up the gui, but closing the gui will no longer shutdown the tray icon.
