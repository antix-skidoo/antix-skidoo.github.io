Shay wrote:Plum forgot that. So I edited after the install.
It's new and we haven't made an official rc1 announcement yet so I imagine many people don't know about it.

Historically, wicd had been disabled by default on the live system  but enabled by default on installed systems. Now the installed system follows the choice on the live system and your selection on a live-usb is sticky so you only have to use the wicd cheat once.   If this new way causes a lot of trouble, we can go back to the old way.   When we were trying to debug the networking problems, it seemed like a good idea to make the networking on the installed system be the same as the networking on the live system to avoid problems but I don't think that's needed now.

Another possibility would be to ask if the user wants to enable wicd (on installed or live-usb systems) if they use the wicd application and wicd is not already enabled.  Or we could just automatically enable it if the wicd application is used.

We added back the"mount=all|usb|off" cheat on the live system to mount all partitions or just usb partitions at boot-time.  This does not carry over to the installed system.  This too is sticky on the live-usb which can be a little confusing at first.   We also enabled the automount option on core systems.  The fstab="auto|hotplug|off" cheat will switch over to the non-X-windows automounting used by default on the core system.  The hotplug mode will cause us to add fstab entries and /media/ directories when a drive is added but we won't actually mount it.  But it is easy to mount with the mount command.  This mode is the default on core systems.
