Either I have missed the note about the signature for these public keys, or we do not have them yet. If they ARE available and I have simply glossed over reading the note, would someone be kind enough to reply with a link to the information about establishing these keys for our repositories (or describe how to do it here)? I'd certainly appreciate it; thanks!

EDIT: Answering my own question, I found the answer: Anticapitalista, in a thread a few weeks ago, asked us if we had any repo-related issues. There were responses indicating that the signature for the public keys for the antiX-specific repos were not there. Anti responded with the solution.

He said:
To fix key for antiX repo:

Type: gpg --keyserver subkeys.pgp.net --recv-keys 3465FC9B
Press: enter / return
Type: gpg --armor --export 3465FC9B | apt-key add -
Press: enter / return
Type: apt-get update
Press: enter / return
Install your applications using your package manager or typing apt-get install PACKAGE NAME

Anti, this site takes care of the slavinko repo. English readers should use the Google translation to see what's needed, but actually the original Slavic site has the correct repo and key information; the Google translation messes that up a bit. You can, however, read it in your language, then go back to the original language to get the correct commands, which, incidentally, are:

Open a terminal, then switch to the root user, then run these commands:

Code: Select all

wget -q http://debs.slavino.sk/repo.asc -O- | apt-key add -

Code: Select all

apt-key add repo.asc

Code: Select all

gpg --recv-keys E9C5D7EF
gpg --export --armor E9C5D7EF | sudo apt-key add -
