The from=all boot parameter may help others who are unable to boot a LiveUSB. It will not help with a LiveCD. This boot parameter will cause all partitions to be scanned even partitions on non-removable devices. The problem was triggered because annoying beggar's usb device was reported by the system as non-removable.

We are modifying our software so that by default all usb devices, in addition to all removable devices, are scanned to find the squashfs file. This should eliminate the need for the from=all workaround. In the meantime, if your antiX-13 LiveUSB errors out with the message:

Code: Select all

Fatal Error:
No usb,cd devices found!
then the from=all boot parameter might get around the problem. If it doesn't solve your problem then another possible cause is a missing driver which I can try to help you solve following the steps annoying beggar used above.
