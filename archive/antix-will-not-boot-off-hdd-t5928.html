The cli-installer in my antiX-15-V_386-core-libre is also screwed in cli-installer: on lines 328,329 there is code grub=hd$gdrv1 which will never render into needed /dev/sda (in my case) . Yes this is also visible in announcement"grub installed on (hd0)" (instead of expected"on (/dev/sda)" ) which can be easily oversighted. Replacing this code with grub="/dev/sda" is rough, but quick working solution (beside manually installing grub after installation by using live CD)
