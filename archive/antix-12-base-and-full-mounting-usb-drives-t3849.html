I have found a second way in which automount of USB drives can be done. I recall discussing this with BitJam in the planning stages of antiX-12 but had not realized that it had been implemented. viewtopic.php?f=43&t=3380&hilit=unplugdrive.sh

In ~/.xinitrc-custom is an entry (commented out default) which does the automounting.

At the time, zenity was a default app in the distro and the command included the parameter"--info-on-mount" to provide on-screen feedback. As zenity is no longer included as a default app it is probably best to edit the command, unless you want install zenity of course.

Again, the original made use of udisks which is now been replaced by udevil so another reason to update the command.

My version of the command is:

Code: Select all

which devmon &>/dev/null && which udevil &>/dev/null && devmon &
So now there are two ways to automatically mount a USB device. The main difference between them is that renaming etc/udev/rules.d/99usb-storage.rules requires root privileges, editing ~/.xinitrc-custom can be done as an unprivileged user. There are other small differences but the main outcome is that both allow automatic mounting and unmounting as a standard user for those who choose to use it.
