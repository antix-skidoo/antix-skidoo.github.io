It would probably be easier to move the antiX scripts and programs to 13.2, then to try and move xorg and the like to version 16.
Most likely they should be in debs in the antiX repo in which case you may be able to Purge the 13.2 debs and scripts followed by installing the version 16 debs. Myself I would load / extract / mount the version 16 image and rsync / copy the scripts you like from usr/local/bin, etc/, skel, lib, etc to the same location in 13.2. Likely most will work but not the startup / live / usb stuff as far as I know. It would also mean manually updating every time there is a package update. It might also be worth exploring options like mounting usr/local/bin from version 16 in version 13.2 and copying over only the configuration files. That way you could update v16 and at the same time (hopefully) update the antiX scripts in the 13.2 install
