1) Stop the display-manager antiX uses from starting in the default runlevel.
For antiX the display-manager is probably slim or lightdm
If you use sysv-init system use sysv-rc-conf to stop it from automatically starting during boot, for systemd use systemctl


2) Install the servers you want to run.

3) Instead of doing 1) you could also apt-get purge all of X and and all apps you don't need, but for a quick comparison that seems to be too much work.

4) I wouldn't think of antiX as a server distribution, but perhaps i look at it the wrong way. I think there is a small installation iso, and i would go for that one (one without X).

A less friendly answer is that you don't want to run a server if you have to ask such questions.
