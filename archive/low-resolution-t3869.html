when you boot up go to tty2 by pressing control + alt +f2
log in as root and type sgfxi, that should but a proper xorg.conf, edit menu.lst and install a proprietary driver if available

if you would like to do similar without installing the driver run:
service slim stop
then
X -configure
then
cp / root / xorg.conf.new / etc /X11 /xorg.conf
then
nano /boot /grub /menu.lst
and remove the noxorg cheat
and finally reboot

Did you try the seccond half?

Maybe you will need to add a few mode lines to the generated xorg.conf.
open / etc/ X11/ xorg.conf as root
menu --> run
type: gksu leafpad / etc/ x11/ xorg.conf | press enter (without the forum required spaces __{{emoticon}}__ )
type: YOUR ROOT PASSWORD | press enter
press: control + f
type: Section"Screen" | press enter
press: control + f
type: SubSection"Display" | press enter
press: control + f
type: Depth | press enter
add below every result of Depth: Modes"1280x1024" #(1280x1024 is your desired / supported resolution)

Restart X, though for good measure I reboot the computer __{{emoticon}}__

Ok not knowing how you had it modified before, here is how I have always modified it, if you know the higher resolution from before then replace 1280x1024 with it
Example using your inxi output from the liquorix kernel:
1280x1024 --> 1440x900
You can add the refresh rate to by adding @refresh rate
Example:
1280x1024 --> 1440x900@60.0
This has not worked 100% of the time for me though so it would be something to try if the first example does not work.
