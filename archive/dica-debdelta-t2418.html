O pacote debdelta (apt-get install debdelta) é muito útil tanto para economizar banda de Internet quanto para apressar as atualizações do sistema.
O comportamento padrão do apt é o de baixar todo o pacote quando este é atualizado, mesmo que a mudança entre as versões seja pequena. Por exemplo: o gnome-icon-theme tem um volume de uns 9 MB e é regularmente atualizado pois muitos programas dependem dele e quando são modificados o gnome-icon-theme também o é. Assim o apt baixa toda vez 9 MB pra atualizar às vezes poucos ícones.
O programa debdelta lê o cabeçalho dos pacotes e baixa somente as mudanças, criando o pacote novo a partir do pacote antigo (não há falhas no processo pois a soma de verificação é conferida). Assim é evidente que baixa-se muito menos dados para se atualizar o sistema.
Normalmente a sequência de atualização é apt-get update && apt-get dist-upgrade; com o debdelta ela passa a ser apt-get update && debdelta-upgrade && apt-get dist-upgrade.
O comando debdelta-upgrade é o que gera os deltas dos pacotes a serem atualizados (a diferença entre o pacote em uso e o novo).

O antiX tem muitos arquivos (principalmente documentação) apagados pra que o iso fique num tamanho menor e assim o debdelta não funciona com esses pacotes que foram em parte apagados (por isso também não funciona se o localepurge estiver instalado), mas à medida que o sistema vai sendo atualizado os pacotes passam a estar na sua versão completa e o debdelta passa a funcionar com eles.
Hoje mesmo eu atualizei o sistema e teriam de ser baixados 95 MB de dados mas o debdelta economizou 50 MB.

Pra que o smxi use o debdelta por padrão*, crie um arquivo /usr/local/bin/smxi-values

Code: Select all

# touch /usr/local/bin/smxi-values
e ponha B_USE_DEBDELTA=true nele

Code: Select all

# echo B_USE_DEBDELTA=true > /usr/local/bin/smxi-values
Se quiser apenas experimentar o debdelta no smxi sem configurá-lo como o padrão, execute o smxi assim

Code: Select all

smxi -! 40
* instale o debdelta primeiro.
