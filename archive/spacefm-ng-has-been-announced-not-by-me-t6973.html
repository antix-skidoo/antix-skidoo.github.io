Hello to all,

This repo of SpaceFM is no longer in active development (you're still welcome to use this issue tracker).
I just want to bring attention to a fork which is being worked on by @Teklad called spacefm-ng.

He has forked from SpaceFM 1.0.5+next, and has fixed some of the bugs reported here.
He has also worked on documentation, etc., and has merged some work from others.
I was just made aware of this fork, so I haven't reviewed it in detail, but according to @OmegaPhil it's the strongest fork going.

Also, if you would like to run alpha but with his bugfixes and changes, you might be able to merge his work into alpha fairly easily, as I mentioned in spacefm-ng issue 6. alpha usually runs quite well and has some nice features and improvements - quite a bit of work went into it, but it didn't quite make it to release.

Translators: Please note that SpaceFM's Transifex server is populated with files from alpha (I just pulled the updates into alpha today - thanks for the work!) These are missing many strings used in next, so these files won't work well with next (or with Teklad's master branch). However, it is possible to merge po files - there are tools to do that. I also discuss this in spacefm-ng issue 6.
