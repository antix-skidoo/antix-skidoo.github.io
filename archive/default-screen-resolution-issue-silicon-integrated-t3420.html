As was mentioned in the Mepis thread anti linked to, you might be better off changing the dpi (dots per inch) instead of the resolution. This can make all the text on the screen larger (or smaller) without affecting the aspect ratio or the"crispness" (assuming 1200X800 is the native resolution of your display device).

On a liveCD/DVD, this can be set when booting with the dpi= boot parameter. For an installed system, you have to edit a config file. Go to:

Code: Select all

control center
    --> system
        --> edit config files
            --> slim.conf
Edit the xserver_arguments line, changing it from:

Code: Select all

xserver_arguments -nolisten tcp
to

Code: Select all

xserver_arguments -dpi 110 -nolisten tcp
The larger the number following"-dpi", the larger your fonts will be. The default is 96. You have to reboot or restart the X-server for this change to take effect. If there are other/different arguments on that line, just leave them alone and simply add"-dpi 110".

You might be able to set the dpi immediately from the command line with:

Code: Select all

# xrandr --dpi 110
When I tried that just now in VirtualBox I got an error message:"failed to get size of gamma ...". YMMV.

Of course, choose a value that works best for you. I think 110 is a good first stab if you want larger fonts.
