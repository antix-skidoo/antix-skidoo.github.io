Cuttlefish:

Can you mount your USB key on a running system?
You should see a antiXlive partition and a second partition antiXdata
Under (your aniXlive mountpoint)/boot/grub you will find a menu.lst file.
Can you post the content of the file?

Then with the USB-live key mounted (or not), type the command:

Code: Select all

blkid
this will print the UUID of the partitions
The output line for antiXlive partition, say sda1 or sdb1..., should match the UUID in menu.lst: rootfs=UUID=xxxxxxx
where xxxxx is, I presume, e4c3b6bf-b325-4a9b-8be-0404288bf12

I am not sure new_usb.tar.gz will help you, or do a better job
It is a command line shell script
Copy the archive somewhere. In that directory, decompress the archive

Code: Select all

tar zxvf new_usb.tar.gz
You get a new_usb.sh script
Make is executable
run:

Code: Select all

./new_usb.sh --help
will display a help message

The above is wrong. Hence:


Error: -b unrecognised option
Create a fromiso (frugal) installation on a USB device
Options (most of them are... optional)
-v print version and exit
-d debug code execution
-h print this help
-I path-to-iso iso file name to be installed
-D device name of USB device where to install the live system
-n do not format partition (use existing first partition)
-f vfat|ext2|ext3 format bootable partition with given filesystem
-s size size of the first partition (in MB)
-F vfat|ext2|ext3 format a second partition with given filesystem
-l label label for the first (bootable) partition
-t title title of the first entry in grub menu
-c"cheat codes" cheat codes appended to first kernel entry in grub menu
-L label label for the second (user data) partition
-T title title of the second entry in grub menu
-C"cheat codes" cheat codes appended to second kernel line in grub menu
-p add cheat codes to enable persistent home and root file systems

Simple usage:
antix2usb.sh -I /path/to/antix-iso.iso -D /dev/sdx
This will erase all data on device
Use it at your own risk, it comes without any warranty.
