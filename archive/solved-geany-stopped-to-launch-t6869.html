That result will occur (expected to occur) when one instance of an app, launched by non-admin user, is already running
and a sudo-permissioned operation attempts to use that running instance instead of starting a new one.
For example: if you launch a filemanager"as root"... then attempt to drag/drop from filemanager to geany window
(or vice versa)

You bumped into it using geany, but same result will occur with other apps as well.

To clear the immediate problem, close all running instances of geany and retry
(idunno whether you dragged onto a geany window or double-clicked...)
If the problem remains, logout and retry after starting a new desktop session.
