Just ran a quick compile to give it a very brief glance...
Downloaded the lib and the environment:
ran: tar -xf edelib*
ran: tar -xf ede*
cd: edelib*
ran: ./configure
found it needed jam and some other things
ran: apt-get -f install jam, and some other things
ran: ./configure
ran: jam
ran: jam install
cd: ../ede*
ran: ./configure
ran: jam
ran: jam install
Then because I wanted to test this as a clean user....
ran: adduser test --force-badname
pressed: Control + alt + f2
logged in as test
ran: startx startede

Seems very clean and smooth looking and based off the source of another window manager that has been around for a while. However at the moment there are a few issues that are really bothersome to me (Note: I have not read through any documentation of any sort). They are some programs do not actually seem to do anything (like ede-exit does not logout)... and the overwhelming fact that there is no window boarders or windows tabs/lists in the panel... so no way to minimize via mouse and even the standard keyboard shortcuts (alt + f4 does not close). This was very troublesome when I opened the help docs and found I could not move, close, resize my maxed iceweasel window. __{{emoticon}}__ There are a few other things, but I do not want to list / go into detail with to many on my part until at least viewing the documentation.

With the newer desktop handling that is going into antiX 14 hopefully alot of the benifits seen in this lightweight desktop environment will be seen in the new antiX. Things like xdg directory / autostart / menu stuff, desktop integration (ideally you only need to change the preferred terminal in 1 area for it to show in the whole menu / control center / toolbar / etc), session logout / switching, and others should show up in the iso for testing at some point. I would like to look at ede in more detail to see if it is taking a similar path... maybe there will be some influencing apear __{{emoticon}}__
